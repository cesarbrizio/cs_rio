# CS RIO — Contexto do Frontend

Resumo do contexto relevante para o app mobile e para as decisões de UX, mapa e experiência do jogador no repositório `cs_rio`.

## Direção do produto

- crime RPG mobile ambientado no Rio de Janeiro
- foco em experiência jogável, não em dashboard administrativo
- mapa, HUD e clareza de feedback como pilares do app

## Marcos que importam para o frontend

### Concepção

- o projeto saiu de uma ideia inicial web e foi redirecionado para mobile nativo
- a stack do app consolidou Expo + React Native + TypeScript

### Experiência do jogador

- `JOGO.md` virou o contrato macro das mecânicas
- `MAPA.md` consolidou a direção visual e espacial
- `PLANO_CORRECAO_UX.md` consolidou o plano de correção da camada de interface

### Estrutura atual

- este repositório concentra apenas o frontend
- o backend operacional foi separado para `../cs_rio_api`
- backlog e rollout do app agora vivem em documentos específicos do frontend
